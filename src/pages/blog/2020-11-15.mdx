## プログラミング・技術

### Tailwind CSS

- 気になっていた [Tailwind CSS](https://tailwindcss.com/) を試した
  - 体験は Chakra UI に近い (Chakra UI が Tailwind を意識して作られているのでそれはそうなのだけど)
  - コードを書く速度は Chakra より Tailwind の方が出る気がする
  - class で styling していくので、「この element にはこの class しか指定できないようにしたい」「特定の style のみ props から注入できるようにしながら atoms なコンポーネントとして運用したい」といったユースケースをどう実現していけば良いのかまだよく分かっていない
    - Tailwind で style を注入できる ≒ primitive / atoms が本来担う部分を達成できていると考えるべき？
    - utility first なライブラリで依存の注入を考えることが間違い？
    - variant を使えばここらへんの問題は解決する？
  - もうちょい書いてみて要所がつかめたらちゃんと記事にまとめたい
